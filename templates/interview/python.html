Question: Explain the difference between lists and tuples in Python.

  • Lists are mutable, ordered collections of elements enclosed in square brackets []. They can be modified after creation.
  • Tuples are immutable, ordered collections of elements enclosed in parentheses (). They cannot be changed once created.

Question: How do you reverse a list in Python?

  • You can reverse a list using the `list.reverse()` method, which modifies the list in place.
  • Alternatively, you can use slicing: `reversed_list = my_list[::-1]` to create a new reversed list.
  • You can also use the `reversed()` function, which returns an iterator: `reversed_list = list(reversed(my_list))`.

Question: What does the `enumerate()` function do in Python?

  • `enumerate()` adds a counter to an iterable and returns it as an enumerate object, which can be used in loops to retrieve both the index and the value.
  • Example: `for index, value in enumerate(my_list):` allows you to iterate with both index and value.

Question: How do you merge two dictionaries in Python (Python 3.9+)?

  • In Python 3.9 and later, you can merge two dictionaries using the `|` operator: `merged_dict = dict1 | dict2`.
  • This creates a new dictionary containing the merged contents of both dictionaries.
  • Alternatively, you can use `dict1.update(dict2)` to merge `dict2` into `dict1`, modifying `dict1` in place.

Question: Explain the difference between `break`, `continue`, and `pass` in Python.

  • `break` exits the nearest enclosing loop entirely.
  • `continue` skips the rest of the code inside the loop for the current iteration and moves to the next iteration.
  • `pass` does nothing and is used as a placeholder where a statement is syntactically required but no action is needed.

Question: What is the purpose of the `assert` statement in Python?

  • The `assert` statement is used for debugging purposes, allowing you to test if a condition in your code returns `True`.
  • If the condition is `False`, `assert` raises an `AssertionError` with an optional message.
  • Example: `assert x > 0, "x must be positive"` ensures that `x` is positive.

Question: How can you check if a file exists in Python?

  • You can check if a file exists using the `os.path.exists()` function from the `os` module.
  • Example: `import os`
    `file_exists = os.path.exists('file.txt')`
  • Another method is to use `Path` objects from the `pathlib` module: `from pathlib import Path`
    `file_exists = Path('file.txt').exists()`.

Question: What is the purpose of the `zip()` function in Python?

  • The `zip()` function combines elements from multiple iterables (like lists or tuples) into tuples, where each tuple contains one element from each iterable.
  • Example: `zip([1, 2], ['a', 'b'])` results in `[(1, 'a'), (2, 'b')]`.
  • `zip()` stops when the shortest input iterable is exhausted.

Question: How do you create a virtual environment in Python?

  • You can create a virtual environment using the `venv` module: `python -m venv myenv`.
  • To activate the virtual environment:
    Windows: `myenv\Scripts\activate`
    Unix/MacOS: `source myenv/bin/activate`.
  • To deactivate, simply run `deactivate` in the terminal.

Question: How do you handle exceptions in Python?

  • Exceptions in Python are handled using `try`, `except`, `else`, and `finally` blocks.
  • `try` contains the code that might raise an exception.
  • `except` catches and handles the exception.
  • `else` is executed if the `try` block does not raise an exception.
  • `finally` runs whether an exception is raised or not, and is used for cleanup actions.

Question: How do you remove duplicates from a list in Python?

  • One way to remove duplicates is to convert the list to a set and back to a list: `unique_list = list(set(my_list))`.
  • However, this method does not preserve the order. To remove duplicates while preserving order, you can use a loop or a dict: `unique_list = list(dict.fromkeys(my_list))`.

Question: How can you generate random numbers in Python?

  • You can generate random numbers using the `random` module. For example, `random.randint(a, b)` generates a random integer between `a` and `b` (inclusive).
  • `random.random()` generates a random float between 0 and 1.
  • `random.choice(sequence)` returns a random element from a non-empty sequence like a list or tuple.

Question: What is the difference between `deepcopy` and `copy` in Python?

  • `copy.copy()` creates a shallow copy of an object, meaning it only copies the object's reference to nested objects, not the objects themselves.
  • `copy.deepcopy()` creates a deep copy of an object, meaning it recursively copies all objects found, producing a fully independent clone.

Question: How does Python's Global Interpreter Lock (GIL) affect multi-threading?

  • The GIL is a mutex that protects access to Python objects, preventing multiple native threads from executing Python bytecodes at once.
  • This means that Python multi-threading does not achieve true parallelism in CPU-bound tasks but can be effective in I/O-bound tasks.

Question: Explain the concept of Python decorators.

  • Decorators are functions that modify the behavior of another function or method.
  • They are often used for logging, access control, memoization, and instrumentation.
  • A decorator takes a function as input, wraps it in another function that extends its behavior, and returns the wrapper function.

Question: What are generators in Python, and how do they differ from normal functions?

  • Generators are functions that return an iterator and allow for lazy evaluation.
  • They are defined with the `yield` keyword, which pauses the function's execution and allows it to be resumed, returning a value to the caller.
  • Unlike normal functions that return a single value, generators return an iterable sequence of values, one at a time, upon each call to `next()`.

Question: What are metaclasses in Python, and why would you use them?

  • Metaclasses are classes of classes that define how a class behaves.
  • They are used to modify or extend the behavior of classes during their creation.
  • Common use cases include implementing singleton patterns, logging, or enforcing design constraints.

Question: How do you prevent a Python class from being instantiated?

  • To prevent a Python class from being instantiated, you can raise a `NotImplementedError` in the `__init__` method.
  • This approach is often used in abstract base classes where instantiation is not desired but inheritance is intended.

Question: Explain the difference between `staticmethod` and `classmethod` in Python.

  • `staticmethod` does not receive an implicit first argument, allowing it to behave like a normal function, except it belongs to the class's namespace.
  • `classmethod` receives the class as the first argument (usually named `cls`), allowing it to modify class state that applies across all instances.

Question: What is the purpose of the `__new__` method in Python?

  • `__new__` is a static method responsible for creating and returning a new instance of a class.
  • It is called before `__init__` and is typically used when subclassing immutable types like `int`, `str`, or `tuple`.

Question: How does Python's memory management work?

  • Python uses a private heap to manage memory, where all objects and data structures are stored.
  • Memory management is handled automatically by Python's garbage collector, which uses reference counting and a cyclic garbage collector to free memory.
  • The `gc` module can be used to interact with the garbage collector.

Question: What are Python's `@property` and `@setter` decorators, and how are they used?

  • The `@property` decorator allows you to define a method that acts as a read-only attribute, providing an interface to instance attributes.
  • The `@setter` decorator is used to define a method that sets the value of a property, allowing for validation or processing when an attribute is modified.
  • They enable encapsulation and control over how attributes are accessed or modified.

Question: What is the difference between `==` and `is` in Python?

  • `==` checks if the values of two objects are equal, meaning it compares their data.
  • `is` checks if two references point to the same object in memory, meaning it compares their identities.

Question: How does the `super()` function work in Python?

  • `super()` is used to call a method from a parent class, typically inside a method that overrides a parent method.
  • It is commonly used in multiple inheritance to ensure that the correct method resolution order (MRO) is followed.

Question: What are `__slots__` in Python, and how do they affect performance?

  • `__slots__` is a mechanism that restricts the creation of instance attributes to a predefined list, preventing the creation of `__dict__` and reducing memory overhead.
  • Using `__slots__` can significantly improve memory efficiency, especially in scenarios with a large number of instances.

Question: Explain the difference between `@staticmethod`, `@classmethod`, and an instance method.

  • Instance methods take `self` as the first argument and can access and modify the instance's state.
  • `@classmethod` methods take `cls` as the first argument, can access and modify class state, and are bound to the class, not the instance.
  • `@staticmethod` methods do not take `self` or `cls` and cannot modify either instance or class state; they are simple utility functions.

Question: What are Python's `weakref` objects, and when would you use them?

  • `weakref` objects are references to objects that do not increase the reference count, allowing the object to be garbage-collected.
  • They are useful for caching or referencing objects without preventing their deletion, reducing memory leaks.

Question: How can you implement a singleton pattern in Python?

  • A singleton pattern ensures that a class has only one instance and provides a global point of access to it.
  • This can be implemented using the `__new__` method, metaclasses, or a decorator that controls instance creation.

Question: What is the purpose of the `with` statement, and how does it work?

  • The `with` statement is used for simplifying exception handling and resource management by ensuring that resources are cleaned up after use.
  • It works with context managers, which implement the `__enter__` and `__exit__` methods to set up and tear down resources.

Question: How do you achieve method overloading in Python?

  • Python does not natively support method overloading, but it can be mimicked using default arguments, variable-length arguments (`*args`, `**kwargs`), or conditional logic within a method.
  • You can also use function decorators to manage different types of input.

Question: What are Python's `dataclasses`, and how do they simplify class creation?

  • `dataclasses` are a module in Python that provide a decorator and functions for automatically generating special methods like `__init__`, `__repr__`, and `__eq__` for user-defined classes.
  • They simplify the creation of classes by reducing boilerplate code, especially for classes that primarily store data.

Question: How does Python's `multiprocessing` module differ from `threading`?

  • `multiprocessing` creates separate memory space and process for each task, achieving true parallelism and avoiding the GIL's limitations.
  • `threading` runs tasks in the same memory space, limited by the GIL, and is better suited for I/O-bound tasks.
